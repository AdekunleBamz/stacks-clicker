# Bug Report Template

Use this template when reporting bugs to ensure all necessary information is included.

## Bug Summary

**Title:** Brief, descriptive summary of the issue.

**Description:** What broke, where it happened, and whether it blocks on-chain actions.

## Environment

- **Network:** (mainnet / testnet / devnet)
- **Wallet:** (Hiro / Leather / other)
- **Browser:** (Chrome / Firefox / Safari / other)
- **OS:** (Windows / macOS / Linux / iOS / Android)

## Steps to Reproduce

Keep the path short and include wallet/network context:

1. Step one
2. Step two
3. Step three

## Expected Result

Describe the intended clicker, tip, or poll behavior.

## Actual Result

What actually happened instead.

## Evidence

Attach the following when available:
- [ ] Screenshots of the issue
- [ ] Console errors (copy full error text)
- [ ] Transaction IDs (from Stacks Explorer)
- [ ] Network request logs

## Impact Scope

Note whether only one wallet type or network is affected:
- [ ] All users affected
- [ ] Specific wallet type affected
- [ ] Specific network affected
- [ ] Edge case / rare condition

## Regression Information

Include the last known good commit hash when the regression window is known:
- **Last known good commit:** (git hash)
- **First bad commit:** (git hash, if known)
