# Documentation Style

- Keep runbooks short and action-oriented.
- Prefer checklists for repeatable operational tasks.
- Include exact command examples when possible.
- Update docs in the same change as behavior updates.
- Replace outdated contract or component names during doc updates.
- Prefer short section headings that reflect the task outcome instead of implementation detail.

## Maintenance Note
- Revisit terminology consistency examples when product naming conventions change.

### Writing Reminder
- Prefer stable relative links when referencing local runbooks from operations docs.
